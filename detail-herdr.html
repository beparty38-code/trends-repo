What is it?

모든 코딩 에이전트를 하나의 터미널에서 실행하고, 누가 막혀있고/작업 중이고/완료됐는지 한눈에 볼 수 있게 해주는 에이전트 멀티플렉서입니다.

How it Works

Server-Client Architecture

백그라운드 서버가 세션을 유지하고, 여러 클라이언트가 접속하는 구조입니다. tmux와 유사하지만 에이전트에 특화되었습니다.

  1. herdr 실행 → 백그라운드 서버 시작 (또는 기존 세션 연결)
  2. 워크스페이스/탭/패널 생성 및 관리
  3. 각 패널에서 코딩 에이전트 실행
  4. 서버가 세션 유지 → detach해도 에이전트 계속 실행
  5. 다시 herdr 실행 → 기존 세션에 재연결

Agent Status Tracking

상태의미
● Blocked에이전트가 사용자 입력을 기다리는 중
● Working에이전트가 작업 수행 중
● Done작업 완료
● Idle대기 상태

Key Features

  • 진정한 터미널 — 각 에이전트가 시뮬레이션이 아닌 실제 터미널 획득
  • 세션 지속성 — 연결 해제해도 백그라운드에서 계속 실행
  • 원격 접속 — herdr --remote로 VPS 서버 제어
  • 소켓 API — 프로그래밍 방식으로 herdr 제어 가능
  • 플러그인 시스템 — 사용자 정의 확장
  • 마우스 네이티브 — 클릭, 드래그, 분할 모두 지원
  • 15+ 에이전트 자동 감지 — 프로세스 이름 + 출력 휴리스틱

Installation

# Quick install
curl -fsSL https://herdr.dev/install.sh | sh

# Homebrew
brew install herdr

# From source
git clone https://github.com/ogulcancelik/herdr
cd herdr
cargo build --release
