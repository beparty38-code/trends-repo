TencentCloud/CubeSandbox

7.2k stars Rust / Go / C AI Infra

What is it?

LLM이 생성한 코드를 안전하게 실행하기 위한 고성능 샌드박스 서비스입니다. KVM 기반 microVM으로 60ms 부팅, 5MB 미만 메모리 오버헤드를 달성합니다.

Architecture

CubeSandbox/
├── CubeAPI/          # REST API 게이트웨이 (Rust, E2B 호환)
├── CubeMaster/       # 클러스터 오케스트레이터
├── Cubelet/          # 컴퓨팅 노드 로컬 스케줄링
├── CubeProxy/        # 리버스 프록시 (E2B 프로토콜 호환)
├── CubeVS/           # eBPF 기반 가상 스위치
├── CubeEgress/       # 보안 게이트웨이 (OpenResty)
├── CubeHypervisor/   # KVM 가상화 계층
├── CubeShim/         # containerd Shim v2
├── cubecow/          # Copy-on-Write 스냅샷 엔진
├── cubelog/          # 로깅 시스템
├── sdk/              # Python 클라이언트 SDK
└── web/              # 웹 콘솔 UI

Key Features

  • 하드웨어 수준 격리 — 각 샌드박스가 자체 Guest OS 커널 보유 (Docker 커널 탈출 불가)
  • E2B SDK 호환 — URL 환경변수 하나만 바꾸면 마이그레이션 완료
  • eBPF 가상 스위치 — 커널 레벨 네트워크 격리
  • 자격증명 금고 — API 키가 샌드박스에 진입하지 않음
  • CubeCoW 스냅샷 — 이벤트 레벨 스냅샷, 즉시 복제, 롤백
  • 송신 제어 — 도메인 허용 목록, 즉시 차단, 감사 로그
  • 웹 콘솔 — 브라우저 기반 시각적 관리

How it Works

  1. RustVMM + KVM으로 microVM 생성 (각각 자체 커널)
  2. eBPF 기반 CubeVS가 커널 레벨에서 네트워크 격리
  3. CubeEgress가 L7 도메인 필터링 + 자격증명 주입
  4. CubeCoW가 Copy-on-Write 스냅샷으로 상태 관리
  5. E2B 호환 REST API로 외부에서 제어
